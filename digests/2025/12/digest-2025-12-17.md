### Confucius Code Agent: Scalable Agent Scaffolding for Real-World Codebases
**Link**: https://arxiv.org/abs/2512.10398
**Published**: December 17, 2025

A production-grade software engineering agent built on the Confucius SDK that operates on enterprise-scale codebases. The system features hierarchical working memory for long-context operations, persistent note-taking for cross-session learning, and a meta-agent that automates configuration synthesis through build-test-improve loops. On SWE-Bench-Pro, it achieves 54.3% Resolve@1, surpassing both research-grade and proprietary agents. The SDK addresses the gap between research prototypes and practical deployment by balancing Agent Experience (AX), User Experience (UX), and Developer Experience (DX).

**Key Insight**: Production coding agents require more than just LLM capabilities‚Äîthey need hierarchical memory systems, session persistence, and automated meta-optimization to handle real-world software engineering at scale.

**Why It Matters**: Most coding agents struggle when scaled to massive repositories and long-horizon tasks. This demonstrates a complete architecture for building extensible, enterprise-ready agents that developers can actually deploy in production environments.

### Verification-Guided Context Optimization for Tool Calling
**Link**: https://arxiv.org/abs/2512.13860
**Published**: December 17, 2025

A framework that uses LLMs as hierarchical editors to automatically refine tool documentation and knowledge bases. The system addresses a critical production problem: tool documentation written for humans often misaligns with how LLMs interpret information. VGCO collects real-world failure cases, identifies context mismatches, then performs structure-aware, verification-guided optimization. The approach is particularly effective in industrial settings with hundreds of overlapping tools, achieving significant improvements in accuracy, robustness, and generalization across different LLMs.

**Key Insight**: Tool-calling effectiveness depends less on model capabilities and more on context quality‚Äîhuman-written documentation often confuses LLMs, requiring systematic, automated refinement.

**Why It Matters**: As enterprises deploy agents with access to hundreds of internal tools, manually maintaining aligned documentation becomes impossible. This provides a scalable solution for industrial-scale tool orchestration.

### VLegal-Bench: Vietnamese Legal Reasoning Benchmark for LLMs
**Link**: https://arxiv.org/abs/2512.14554
**Published**: December 17, 2025

The first comprehensive benchmark for evaluating LLMs on Vietnamese legal tasks, featuring 10,450 expert-annotated samples grounded in authoritative legal documents. Built on Bloom's cognitive taxonomy, it assesses multiple levels of legal understanding through real-world scenarios including RAG, multi-step reasoning, and practical assistant workflows. The benchmark addresses challenges posed by Vietnamese law's complexity, hierarchical organization, and frequent revisions.

**Key Insight**: Domain-specific agent evaluation requires cognitively-grounded benchmarks that reflect real-world workflows, not just generic language tasks.

**Why It Matters**: As LLMs are deployed for specialized legal work globally, region-specific benchmarks ensure agents can reliably interpret complex, hierarchical legal frameworks. This model extends beyond Vietnamese law to other specialized domains.
